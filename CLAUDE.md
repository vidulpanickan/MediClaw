## Commands

### Build & Check

```bash
cd nemoclaw && npm run build          # Compile TypeScript plugin → dist/
cd nemoclaw && npm run dev            # Watch mode
cd nemoclaw && npm run check          # lint + format:check + tsc --noEmit (fast validation)
make check                            # Full suite via prek (all pre-commit hooks)
npm run typecheck                     # Type-check root JS files (jsconfig.json)
```

### Test

```bash
npm test                              # All test projects (vitest)
npx vitest run --project cli          # CLI tests only (test/**/*.test.js)
npx vitest run --project plugin       # Plugin tests only (nemoclaw/src/**/*.test.ts)
npx vitest run test/onboard.test.js   # Single test file
npx vitest run --coverage             # With v8 coverage
```

E2E tests require `BREV_API_TOKEN` env var: `npx vitest run --project e2e-brev`

### Lint & Format

```bash
npm run lint                          # ESLint on root JS (bin/, test/, scripts/)
cd nemoclaw && npm run lint           # ESLint on plugin TS (strict type-checked)
cd nemoclaw && npm run format         # Prettier on plugin TS
make format                           # Format both TS and Python
```

### Docs

```bash
make docs                             # Build Sphinx docs (requires uv)
make docs-live                        # Live server with auto-reload
```

### Git Hooks (prek)

Hooks are managed by [prek](https://prek.j178.dev/) via `.pre-commit-config.yaml`. Installed automatically on `npm install`. Manual run: `npx prek run --all-files`.

## Architecture

Four components connected in a pipeline:

```
User CLI (bin/nemoclaw.js)
    → TypeScript Plugin (nemoclaw/src/)
        → Python Blueprint (nemoclaw-blueprint/)
            → OpenShell CLI → Sandbox Container
```

### CLI (`bin/nemoclaw.js`)
Node.js dispatcher. Routes global commands (`onboard`, `list`, `deploy`, `start`, `stop`, `status`, `debug`, `uninstall`) and sandbox-scoped commands (`<name> connect|status|logs|destroy|policy-add|policy-list`). Uses helper modules in `bin/lib/` for credentials, registry, NIM lifecycle, and policies.

### TypeScript Plugin (`nemoclaw/src/`)
Registers with the OpenClaw host API via `register(api: OpenClawPluginApi)` in `index.ts`. Three subsystems:
- **Slash commands** (`commands/slash.ts`): `/nemoclaw status`, `/nemoclaw eject`, `/nemoclaw onboard` inside the chat UI
- **Blueprint runner** (`blueprint/runner.ts`): Orchestrates sandbox lifecycle (plan → apply → status → rollback) by invoking the Python blueprint as a subprocess
- **State** (`blueprint/state.ts`): Persists deployment state to `~/.nemoclaw/state/`
- **Onboard config** (`onboard/config.ts`): Manages inference provider configuration at `~/.nemoclaw/config.json`

### Python Blueprint (`nemoclaw-blueprint/`)
Versioned orchestrator. `blueprint.yaml` defines sandbox image, inference profiles (default/ncp/nim-local/vllm), and network policy. The runner resolves, digest-verifies, and executes this artifact through the OpenShell CLI.

### State & Config Locations
- `~/.nemoclaw/config.json` — Onboard config (endpoint, model, provider, profile)
- `~/.nemoclaw/state/nemoclaw.json` — Deployment state (lastRunId, sandboxName, timestamps)
- `~/.nemoclaw/state/runs/<run_id>/plan.json` — Blueprint plan snapshots
- `~/.nemoclaw/sandboxes.json` — Sandbox registry
- `~/.nemoclaw/credentials.json` — Provider credentials (mode 600)

## Conventions

- **Commits**: Conventional Commits (`feat(scope): description`). DCO sign-off required (`git commit -s`).
- **SPDX headers**: All source files need Apache-2.0 headers. Auto-fixed by: `scripts/check-spdx-headers.sh --fix`
- **TypeScript style**: Strict (`no-explicit-any`, `no-floating-promises`, `consistent-type-imports`, `switch-exhaustiveness-check`). Prettier: double quotes, semicolons, trailing commas, 100 char width.
- **Test naming**: Plugin tests are colocated as `*.test.ts`; CLI tests live in `test/*.test.js`.
- **Vitest projects**: Three projects defined in `vitest.config.ts` — `cli`, `plugin`, `e2e-brev`.
- **Node.js**: Requires >=22.0.0. Root JS uses CommonJS (`bin/`, `test/`); plugin uses ESM.
- **Python**: Use `uv` for dependency management (not pip).
- **Docs**: Source of truth is `docs/`. Generated skills in `.agents/skills/docs/` are autogenerated — never hand-edit.
